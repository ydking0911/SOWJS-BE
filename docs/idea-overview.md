# SOWJS.KR — 서비스 아이디어 개요

> **한 줄 정의**: League of Legends 내전(Custom Game)에서 AI가 개인별 실질 기여도를 분석해 가장 공정한 팀을 자동으로 구성해주는 웹 서비스

---

## 🎮 배경 및 문제 정의

리그 오브 레전드 내전(Custom Game)은 친구들끼리 팀을 짜서 즐기는 가장 인기 있는 게임 방식 중 하나입니다.  
그러나 현재 팀 구성 방식에는 명확한 한계가 있습니다.

### 현재 방식의 문제점

| 문제 | 설명 |
|------|------|
| **티어 불신** | 같은 골드라도 실력 편차가 매우 크다 (골드 1 ≠ 골드 5) |
| **단순 합산 방식** | "Gold 1 + Silver 2 = 팀 A" 식의 단순 계산은 실제 밸런스를 보장하지 못함 |
| **최근 폼 미반영** | 오래된 티어는 현재 실력을 반영하지 않음 (슬럼프, 복귀 등) |
| **포지션 미고려** | 라인 배치 없이 팀을 나누면 특정 팀에 딜러가 몰리는 등 구조적 불균형 발생 |
| **숨은 기여자 미발굴** | KDA 외에 시야, CS, 오브젝트 기여 등 실질 기여도가 반영되지 않음 |

---

## 💡 핵심 아이디어: '내전용 히든 MMR'

단순 티어 합산을 넘어, **실질 승률 기여도**를 분석하는 것이 핵심입니다.

### AI 모델의 역할

Riot API로 수집한 **최근 20~50 게임** 데이터를 기반으로 다음 지표를 종합합니다:

- **KDA** (Kill / Death / Assist)
- **분당 CS** (Creep Score per Minute)
- **시야 점수** (Vision Score)
- **주 라인 승률** (Lane Win Rate)
- **오브젝트 기여도** (Dragon, Baron 참여율)
- **현재 티어 및 LP**

이를 통해 각 유저의 **'내전용 히든 MMR'** 을 산출합니다.

### 최적화 알고리즘

10명의 유저를 입력하면 두 팀의 **예상 승률이 50:50에 가장 가깝게 수렴**하도록 조합을 생성합니다.

- 라인 스왑 가능성까지 고려
- 주 라인 / 부 라인 설정 반영
- 실시간 최근 폼(Performance) 가중치 적용

---

## 🌐 서비스 이름

**SOWJS.KR**

- `.gg` = 게임 커뮤니티에서 "Good Game"을 의미하는 관용 표현
- 전적 검색 사이트들의 네이밍 컨벤션 활용 (op.gg, u.gg 등)
- 직관적이고 기억하기 쉬운 브랜드

---

## 🗺️ 서비스 구조 (High-Level)

```
사용자 입력 (10명 소환사명)
        ↓
Riot API 데이터 수집 (최근 20~50게임)
        ↓
AI 히든 MMR 산출 엔진
        ↓
팀 밸런싱 최적화 알고리즘
        ↓
최적 팀 구성 결과 출력
        ↓
(부가) 전적 검색 / 메타 분석 / 빌드 추천
```

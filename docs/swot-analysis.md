# SWOT 분석 — SOWJS.KR

> SOWJS.KR 서비스의 강점, 약점, 기회, 위협 요소를 분석합니다.

---

## 📊 SWOT Matrix

|  | **긍정적 요소** | **부정적 요소** |
|--|----------------|----------------|
| **내부 요인** | 💪 Strengths (강점) | ⚠️ Weaknesses (약점) |
| **외부 요인** | 🚀 Opportunities (기회) | 🔴 Threats (위협) |

---

## 💪 Strengths (강점)

### S1. 차별화된 핵심 기술: '히든 MMR' 알고리즘
- 단순 티어 합산이 아닌 **실질 기여도 기반** 팀 구성
- KDA, CS, 시야, 오브젝트 참여율 등 다차원 지표 분석
- 기존 서비스(op.gg, u.gg)가 제공하지 않는 **내전 특화** 기능

### S2. 명확한 타겟 유저와 강한 Pain Point 해결
- "내전 팀 짜기 어렵다"는 문제는 수백만 LoL 유저가 공감하는 실제 불편함
- 솔루션이 직관적이고 즉각적인 가치를 제공

### S3. Riot API 공식 지원
- Riot Games의 공개 API를 통해 신뢰도 높은 데이터 수집 가능
- 별도 데이터 수집 인프라 없이 실시간 데이터 활용

### S4. 낮은 진입 장벽
- 소환사명만 입력하면 즉시 사용 가능 (회원가입 불필요한 MVP)
- 디스코드/카톡방 복붙 인식으로 UX 마찰 최소화

---

## ⚠️ Weaknesses (약점)

### W1. Riot API 의존성
- API 정책 변경, 호출 제한(Rate Limit), 서비스 중단 시 직접적 영향
- API 키 등급에 따라 데이터 수집량 제한 (Production Key 필요)

### W2. 알고리즘 신뢰도 확보의 어려움
- "AI가 짜준 팀이 왜 불공평하냐"는 유저 불만 가능성
- 히든 MMR 산출 로직의 투명성 부족 시 신뢰도 하락

### W3. 초기 트래픽 확보의 어려움
- LoL 커뮤니티는 이미 op.gg, u.gg 등 강력한 경쟁자 존재
- 신규 서비스에 대한 인지도 구축에 시간과 비용 필요

### W4. 소규모 팀 개발 리소스 한계
- 지속적인 메타 업데이트, 패치 대응, 서버 유지 비용
- 챔피언 빌드/스킬 순서 등 콘텐츠 업데이트 부담

---

## 🚀 Opportunities (기회)

### O1. 거대한 시장 규모
- LoL은 전 세계 월간 활성 유저 1억 명 이상
- 한국은 세계 최고 수준의 LoL 인구 밀도

### O2. 내전 문화의 성장
- 유튜버, 스트리머 내전 콘텐츠 인기 급상승
- 아마추어 e스포츠 대회 증가 추세

### O3. 디스코드 봇 생태계 확장
- 게이머 커뮤니티의 디스코드 의존도 증가
- 봇 구독 모델로 B2B 수익화 가능

### O4. AI/ML 기술 트렌드
- AI 기반 게임 분석에 대한 유저 관심 증가
- "AI가 추천한 팀"이라는 마케팅 포인트로 활용 가능

### O5. 타 게임으로의 확장 가능성
- 발로란트, TFT 등 Riot Games 타 게임으로 서비스 확장
- 타 게임사 API 연동으로 멀티게임 플랫폼화

---

## 🔴 Threats (위협)

### T1. Riot Games의 직접 경쟁 또는 정책 제한
- Riot이 유사 기능을 공식 클라이언트에 내장할 가능성
- API 이용 약관 위반 시 서비스 중단 위험

### T2. 강력한 기존 경쟁자
- op.gg, u.gg, blitz.gg 등이 유사 기능 추가 가능
- 대형 서비스의 브랜드 파워와 기존 유저 기반

### T3. 개인정보 및 데이터 이슈
- 유저 게임 데이터 수집/분석에 대한 거부감
- GDPR, 개인정보보호법 준수 필요

### T4. 메타 변화 속도
- LoL 패치 주기(2주)마다 챔피언 밸런스 변화
- 지속적인 알고리즘 및 데이터 업데이트 필요

---

## 🎯 SWOT 전략 도출

### SO 전략 (강점 × 기회 활용)
- 히든 MMR 알고리즘을 핵심 마케팅 포인트로 활용하여 내전 커뮤니티 공략
- 디스코드 봇 연동으로 게이머 커뮤니티 내 바이럴 확산

### ST 전략 (강점으로 위협 대응)
- 알고리즘 투명성 공개(지표 설명)로 신뢰도 확보, 경쟁자 대비 차별화 유지
- Riot API 외 자체 데이터 수집 방안 병행 연구

### WO 전략 (약점 보완 × 기회 활용)
- MVP 단계에서 핵심 기능만 집중 개발, 빠른 시장 진입으로 선점 효과
- 커뮤니티 피드백 기반 알고리즘 개선으로 신뢰도 점진적 구축

### WT 전략 (약점 × 위협 최소화)
- Riot API 의존도를 줄이기 위한 캐싱 전략 및 자체 데이터 레이어 구축
- 법적 리스크 최소화를 위한 이용약관 및 개인정보처리방침 명확화
